solo game + mode #3

Suppose players P1 and P2 have been matched together. This is the moment where match_id is created and match timer begins.

Start game

The game should be started only if entry fee is collected from both players.

In the case that P2 does not send the entry fee, the game won’t start and P1 will be refunded after maxMatchDuration timeout.

Scenario 1: both players finish game normally

First, consider the scenario where both player finish the game normally.

Scenario 2: one player fails to finish the game

Second, consider the scenario where P2 finishes the match normally but P1 fails to finish the game, which is due to either of following reasons.

Scenario 2.1: P1 actively aborts the game by clicking the “abort” button.

In this case, CelerX server will declare P2 as the winner immediately after P2 submits his game score. Winning reason: P1 abort.

Scenario 2.2: P1 fails to submit score before maxMatchDuration timeout (e.g., P1 lost cconnection or P1 is an extremely slow player).

In this case, CelerX server will declare P2 as the winner after maxMatchDuration timeout. Winning reason: P1 timeout.

Scenario 3: both players fail to finish the game

Finally, consider the scenario where both P1 and P2 fail to finish the match (i.e., both players abort the game or both players fail to submit score before maxMatchDuration). In this case, the match is a draw and entry fee will be returned to both players.

Scenario 3.1 Both players actively abort the game.

Scenario 3.2 Both players exceed maxMatchDuration timeout.

Exchange state (optional)

In some use cases, players may need to exchange auxiliary information even in solo game competitions (e.g., a player may want to see his opponent’s real-time game score).

Note that state exchange is optional for solo game and signature is not required.
